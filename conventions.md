# Conventions

Here you can read about our code conventions.

## Language

Everything in this project is in english.

## Liquid

In liquid we use [camalCasing](https://developer.mozilla.org/en-US/docs/Glossary/Camel_case)

## CSS

In CSS we use [kebab-case](https://developer.mozilla.org/en-US/docs/Glossary/Kebab_case) and we nest our code.

## JavaScript

In javascript we use [camalCasing](https://developer.mozilla.org/en-US/docs/Glossary/Camel_case)

## Git

For issues naming we use short descriptive naming with prefixes to describe what use the branch serves.

for example ``[Feature]Comments news page``

### pull request

for pull requests we use a template.

### issues

We make big issues and break them up in smaller issues so that they are bite sized.
